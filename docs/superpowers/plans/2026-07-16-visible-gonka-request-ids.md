# Visible Gonka Request IDs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the two successful Gonka Request IDs visible by default in the Xiami exhibit while clearly distinguishing the historical verified run from the current live session.

**Architecture:** Store the immutable historical run in a dedicated public JSON artifact instead of embedding provenance in application source. Load it during initialization, render a focused verification record above the current-session council, and preserve the existing live, partial, cached, and fallback state machinery unchanged.

**Tech Stack:** Static HTML/CSS/JavaScript, JSON, Node test runner, Playwright.

## Global Constraints

- Label the record as a historical real Gonka Router call dated `2026-07-16`.
- Display both model names, scores, and complete Request IDs without truncating the underlying text.
- Never present the historical record as a request generated by the current page load.
- Keep `mock_` IDs visibly separate in the current-session council when no live request has run.
- Preserve responsive layout at desktop and mobile widths.
