# 赛博记忆公墓 UI 图片素材需求

## 1. 视觉目标

项目的核心视觉定位是“真实 Web2 历史截图 + 黑暗数字档案馆”。图片需要像可供查证的历史展品，而不是普通的科技感氛围图。

关键词：数字遗产、旧网页、档案柜、服务器、旧显示器、网页快照、记忆废墟、庄重、低饱和、可考据。

避免：AI 紫蓝渐变、抽象光球、普通数据中心图库照片、纯氛围风景、过度模糊、大面积内嵌文字、水印。

## 4. 各案例画面方向

### 4.1 虾米音乐

- 优先素材：歌单封面墙、歌曲播放器、长乐评、精选集页面、音乐人主页。
- 可保留色彩：虾米橙色，但其余区域由项目统一降饱和。
- 避免：只放一个虾米 Logo，或使用与历史界面无关的耳机图库照片。

### 4.2 人人网 / 校内网

- 优先素材：旧版个人主页、新鲜事、相册、日志和留言板入口。
- 时间方向：2008–2012 年的高峰时期界面优先。
- 隐私要求：打码或裁掉真实姓名、头像、学校、联系方式和私人留言。

### 4.3 网易博客

- 优先素材：博客首页、日志文章页、相册和个人主页模板。
- 画面气质：保留早期个人网页的排版和装饰特征，不必追求现代简洁。
- 隐私要求：不保留完整个人文章、账号信息或评论。

### 4.4 天涯社区

- 优先素材：论坛板块列表、长帖页面、回复楼层和旧版导航。
- 画面气质：密集、粗粝、像一处曾经人声鼎沸的公共广场。
- 隐私要求：长帖只保留标题或局部界面，不归档大段用户原文。

### 4.5 猫扑网

- 优先素材：大杂烩、论坛首页、帖子列表和早期门户导航。
- 画面气质：保留早期中文互联网的高密度、亚文化和论坛感。
- 避免：与猫扑社区无关的猫图片、现代社交应用截图或纯 Logo。

## 5. 技术与构图规范

1. 展品卡片统一使用 4:3 画幅。
2. 关键内容放在画面中央 70% 的安全区域，避免被响应式裁切。
3. 图片内不写页面标题、不写按钮文案，文字由 HTML 层渲染。
4. 可保留平台原始 Logo 和界面导航，但不应将 Logo 单独放大作为整张图。
5. 提供原始清晰素材，不要预先加暗角、颗粒、降饱和或复古滤镜。
6. 不使用含有图库水印、媒体大型水印或来源不明的商业摄影。
7. 展示历史网页时，优先保留可识别的原始平台视觉，后续再通过 CSS 统一校色。

## 6. 隐私与版权边界

- 只使用公开网页、官方公告、公开媒体报道或 Wayback Machine 中的历史界面。
- 不使用私密相册、私信、付费内容、非公开账号页面或整篇用户原文。
- 人人网、博客和论坛截图中的真实姓名、头像、学校、电话、邮箱和留言需要模糊或裁切。
- 不将完整新闻文章、完整乐评或长帖复制进项目，只展示必要的界面片段和来源元数据。
- 保留每张历史截图的来源 URL 和获取日期，便于后续写入证据清单。

## 7. 素材交付清单

第一批优先交付：

1. 开屏全屏背景 1 张。
2. 首页数字遗址主视觉 1 张。
3. 虾米音乐历史截图 2–3 张。

第二批：

1. 人人网 / 校内网截图 1–3 张。
2. 网易博客截图 1–3 张。
3. 天涯社区截图 1–3 张。
4. 猫扑网截图 1–3 张。

可接收格式：PNG、JPG、JPEG、WebP。优先提供原图，无需手动压缩或重命名。

如果素材来自网站或 Wayback Machine，请同时保留来源链接。

## 8. 素材接入处理

收到素材后，项目内将统一完成：

1. 裁切和主体安全区校正。
2. 姓名、头像和私人内容模糊。
3. 低饱和、暗角、颗粒和统一色温处理。
4. 转换为 WebP 并压缩文件体积。
5. 生成移动端和桌面端适配版本。
6. 替换案例卡片、墓碑详情和开屏背景的素材引用。
7. 检查替代文字、加载性能和响应式裁切。
8. 重跑桌面端与移动端 Playwright 视觉回归。
